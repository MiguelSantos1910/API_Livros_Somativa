# # 🌐 API REST e RESTful

## O que é uma API?
**API (Application Programming Interface)** é uma interface que permite que sistemas, aplicativos e serviços se comuniquem entre si, trocando dados de forma padronizada.

---

## 🔹 O que é API REST?
**REST (Representational State Transfer)** é um padrão de arquitetura para criar APIs usando o protocolo HTTP.

Uma API REST permite que um cliente (ex: app, site, mobile) se comunique com um servidor usando requisições HTTP, como:

- GET → buscar dados  
- POST → enviar dados  
- PUT / PATCH → atualizar dados  
- DELETE → remover dados  

---

## ⚙️ Características da API REST

- Comunicação via HTTP  
- Usa JSON ou XML para troca de dados  
- Sem estado (stateless) → cada requisição é independente  
- URLs representam recursos (ex: /users, /products)  
- Fácil integração com front-end e mobile  

---

## ✅ O que é API RESTful (Full REST)?
Uma **API RESTful** é uma API que segue **todas as boas práticas do padrão REST**.

Ela é considerada “Full REST” quando:

- Usa corretamente os métodos HTTP (GET, POST, PUT, DELETE)  
- Usa códigos de status HTTP (200, 201, 400, 404, 500)  
- Tem URLs bem definidas (ex: /api/v1/users)  
- Não mantém sessão no servidor (stateless)  
- Retorna dados principalmente em JSON  
- Segue princípios de arquitetura REST  

---

## 📌 Diferença entre REST e RESTful

| REST | RESTful |
|------|---------|
| É o padrão teórico | É a implementação prática |
| Define regras | Segue todas as regras do REST |
| Pode ser parcial | Implementação completa (Full REST) |

---

## 🚀 Tecnologias usadas em APIs REST

- Node.js / Express  
- Python (Flask, FastAPI, Django)  
- Java (Spring Boot)  
- PHP (Laravel)  
- C# (.NET)  

---

## 🎯 Aplicações de APIs REST

- Aplicativos mobile  
- Sistemas web (front-end e back-end)  
- Integração entre sistemas  
- Microserviços  
- IoT e automação  

---

## 📚 Conclusão
APIs REST são essenciais para comunicação entre sistemas modernos, e uma API RESTful segue boas práticas completas, garantindo desempenho, escalabilidade e padronização.
